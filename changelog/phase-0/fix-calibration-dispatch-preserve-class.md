# fix(calibration): preserve input class when calibrating survey objects

**Date**: 2026-03-05
**Branch**: fix/calibration-dispatch-preserve-class
**Phase**: Phase 0

## Changes

- Fix `calibrate()`, `rake()`, and `poststratify()` to preserve the input
  class (`survey_taylor` or `survey_calibrated`) instead of promoting to
  `survey_calibrated`; all three now call `.update_survey_weights()` (already
  used by `adjust_nonresponse()`) rather than the removed `.new_survey_calibrated()`
- Delete `R/constructors.R` — `.new_survey_calibrated()` has no remaining call sites
- Update `.update_survey_weights()` comment in `R/utils.R` to reflect that all
  four user-facing functions now use it
- Extend `test_invariants()` in `helper-test-data.R` with a `survey_taylor` branch
- Rewrite dispatch tests in `test-02-calibrate.R`, `test-03-rake.R`,
  `test-04-poststratify.R`: survey_taylor → expect `survey_taylor`; survey_calibrated
  tests construct input directly via `surveycore::survey_calibrated()` instead of
  chaining through a calibration function
- Fix `test-05-nonresponse.R` survey_calibrated happy-path test to construct
  input directly (was previously piping through `calibrate()` which now returns
  `survey_taylor`, breaking the canary test)
- Update `plans/spec-phase-0.md` and `plans/impl-phase-0.md` to reflect the
  corrected dispatch table, rationale, and removed `constructors.R`
